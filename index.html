Step 1

The Context Layer

AI doesn't know what you're building unless you tell it. Good context turns generic responses into tailored solutions.

Pro Tip

Always include: Project type, target audience, tech stack, and business goals.

Example Comparison

❌ Bad Prompt

"Write a login form"

✅ Excellent Prompt
// Context sets the stage
"I'm building a Next.js 14 SaaS app with TypeScript and Tailwind.
Users are enterprise clients who need multi-factor authentication.
We're using Prisma with PostgreSQL for the database."
Step 2

The Specificity Matrix

Vague requests get vague results. Be surgical with your requirements.

Component-Level Specificity

❌ Vague Request

"Make a button"

✅ Specific Request

"Create a React button component with:
• Primary/secondary/ghost variants
• Small/medium/large sizes
• Loading state with spinner
• Disabled state with opacity
• Icon support (left/right/both)
• Full TypeScript interface
• Tailwind classes for dark mode
• Accessibility attributes
• Storybook stories
• Unit tests with Jest"

Function-Level Specificity

❌ Vague Request

"Write authentication"

✅ Specific Request

"Implement a NextAuth.js configuration with:
• Email/password login
• Google OAuth provider
• JWT session strategy
• Database adapter for Prisma
• Role-based access control
• Session callback to add user role
• Middleware for route protection
• Error handling for each provider"

Step 3

The Constraints Section

Most people forget this critical part. Tell the AI what NOT to do to avoid unwanted outputs.

Why Constraints Matter

  • Prevents AI from making assumptions
  • Ensures compliance with your tech standards
  • Eliminates unwanted features or approaches

Constraints Example

CONSTRAINTS:
- DO NOT use any external UI libraries (build from scratch)
- DO NOT use useState for form handling (use react-hook-form)
- DO NOT add any console.log statements in production code
- DO NOT hardcode API URLs (use environment variables)
- DO NOT use any deprecated Next.js 13 patterns
- MUST be TypeScript strict mode compliant
- MUST follow ESLint rules (no-explicit-any, etc.)
- MUST be mobile-responsive first

Common Mistakes & Fixes

Avoid these pitfalls that waste time and produce poor results.

Mistake: "Make it fast"

Too vague, results in generic optimizations

❌ What you get: Basic suggestions like "use memoization"

✅ The Fix:

"Optimize for O(log n) time complexity, implement debouncing on search, use React.memo for expensive components, lazy load non-critical resources."

Mistake: "Add validation"

No details on what or how to validate

❌ What you get: Basic HTML5 validation attributes

✅ The Fix:

"Implement Zod schema validation with custom error messages for: email format, password strength (8+ chars, mixed case, numbers), unique username, and phone number format."
